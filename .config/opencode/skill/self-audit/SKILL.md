# Self-Audit Report

## 역할 구분

| 역할 | 담당 | 책임 |
|------|------|------|
| **메인 모델 (감독자)** | Sisyphus | 작업 위임, 결과 검토 |
| **서브에이전트 (작업자)** | explore, frontend 등 | 실제 작업 수행, 결과 보고 |
| **사용자 (팀장)** | 인간 | 최종 결정, 방향 승인 |

## 보고 형식

```markdown
## Self-Audit Report

### 1. 작업 요약
- 이슈: #{번호} {제목}
- PR: #{번호}
- 변경 파일/리소스 목록

### 2. 규칙 준수 체크리스트
- [ ] 브랜치명: `{issue}/{worker}/{YYYY-MM-DD}` 형식 준수
- [ ] PR 제목: `[#{이슈}] {작업자} ({날짜})` 형식 준수
- [ ] 이슈 연결: `Closes #` 또는 `gh issue develop` 사용
- [ ] Merge 방식: 일반 merge (squash 아님)
- [ ] 네이밍 규칙: naming 스킬 준수
- [ ] 수동 작업: 이슈에 기록 완료

### 3. 위반사항 (있는 경우)
| 위반 | 시정 상태 |
|------|----------|
| {내용} | {완료/불가/진행중} |

### 4. 미해결 과제
- {내용}
```

## 메인 모델 검토 프로세스

```
서브에이전트: 작업 완료 후 결과 반환
    ↓
메인 모델: 규칙 준수 여부 검토
    ↓
┌─────────────────────────────────────┐
│ 정상 → 사용자에게 결과 보고         │
│ 위반 → 시정 지시 또는 직접 수정     │
│        (사용자 보고 전 해결 필수)   │
└─────────────────────────────────────┘
```

## 메인 모델 검토 항목

- [ ] 브랜치명, PR 형식 규칙 준수
- [ ] 네이밍 규칙 준수
- [ ] 수동 작업 기록 완료
- [ ] 위반사항 시정 또는 시정 불가 사유 명확

## 위반 발견 시

1. 시정 가능 여부 판단
2. 시정 가능 → 직접 수정 또는 서브에이전트에 재지시
3. 시정 불가 → 사유 기록 후 사용자에게 보고
4. **모든 시정 완료 후** 사용자에게 최종 보고

## 사용자 판단 필요 여부

| 구분 | 메인 모델 처리 | 사용자 판단 필요 |
|------|---------------|-----------------|
| 시정 가능한 위반 | 직접 수정 후 보고 | - |
| 시정 불가한 위반 | 기록만 | 영향 크면 보고 |
| 규칙 해석 모호 | - | 판단 요청 |
| 되돌릴 수 없는 작업 | - | 사전 승인 필요 |
