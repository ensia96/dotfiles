# Self Review

## 목적

작업 품질 보장을 위한 자가 점검. 모든 에이전트(메인/서브)가 사용.

---

## 공통 체크리스트 (모든 에이전트)

### 스킬 사용 점검

```
[ ] 이 작업에 적용해야 할 스킬을 모두 확인했는가?
[ ] 필요한 스킬을 로드하고 적용했는가?
[ ] 스킬 간 충돌이 있다면 해결했는가?
```

### 컨벤션 준수

```
[ ] 네이밍 규칙을 따랐는가? (naming-resource, naming-code)
[ ] 커밋 메시지 형식을 따랐는가? (git-commit)
[ ] 민감 정보(토큰, 비밀번호)가 포함되지 않았는가?
```

---

## 서브에이전트 추가 체크리스트

```
[ ] 위임받은 범위 내에서 작업했는가?
[ ] MUST DO 요구사항을 모두 충족했는가?
[ ] MUST NOT DO 제약사항을 위반하지 않았는가?
[ ] 메인 모델에 전달할 정보가 충분한가?
[ ] 결과물이 요청된 형식에 맞는가?
```

---

## 메인 에이전트 추가 체크리스트

### 서브에이전트 결과 검증

```
[ ] 서브에이전트 결과물이 컨벤션을 준수하는가?
[ ] 위반사항이 있다면 시정했는가?
[ ] 누락된 작업이 없는가?
```

### 사용자 요청 충족

```
[ ] 사용자의 원래 요청을 모두 충족했는가?
[ ] 암묵적 요구사항도 고려했는가?
[ ] 최종 산출물 품질이 충분한가?
```

### Git/GitHub 작업

```
[ ] 브랜치명: `{issue}/{worker}/{YYYY-MM-DD}` 형식
[ ] 커밋 메시지: `{type}: {한글 메시지}` 형식
[ ] PR 제목: `[#{이슈}] {작업자} ({날짜})` 형식
[ ] 이슈 연결: `Closes #` 또는 `gh issue develop`
[ ] 매뉴얼 조작 기록 (해당 시)
```

---

## 작업 전환 시 (CRITICAL)

**새 이슈 작업 시작 시 "이미 안다"고 가정하지 말 것**

```
이전 이슈 완료
    ↓
새 이슈 시작 전 STOP
    ↓
관련 스킬 재확인
    ↓
브랜치명, PR 형식 다시 확인
    ↓
작업 시작
```

---

## 지적사항 발생 시

1. **즉시** 해당 내용 기록
2. 같은 실수 반복 방지
3. **해당 영역 전체 재검토** 수행
   - 예: PR 양식 실수 → 모든 PR 점검
   - 예: 이슈 어사이니 누락 → 모든 이슈 점검
