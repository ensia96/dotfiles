# Self Review

## 커밋 전 체크리스트 (매 커밋마다)

```
[ ] 커밋 메시지가 한글인가? (type 제외)
[ ] 형식이 "{type}: {메시지}" 인가?
[ ] 민감 정보(토큰, 비밀번호)가 포함되지 않았는가?
```

## 브랜치 생성 전 체크리스트

```
[ ] 이슈 번호가 맞는가?
[ ] 작업자 이름이 맞는가?
[ ] 날짜가 오늘인가? (YYYY-MM-DD)
[ ] 연도가 맞는가? (현재 연도)
```

## PR 생성 전 체크리스트

```
[ ] 제목이 "[#{이슈}] {작업자} ({날짜})" 형식인가?
[ ] 어사이니가 지정되었는가?
[ ] 이슈가 연결되었는가? (Closes # 또는 gh issue develop)
[ ] 변경사항 요약이 있는가?
[ ] 매뉴얼 조작 기록이 있는가? (해당 시)
```

## 작업 완료 전 필수 검토

- [ ] 컨벤션 준수 여부 확인
- [ ] 커밋 메시지 형식 검토
- [ ] 민감 정보 노출 여부 확인
- [ ] 정렬 규칙 준수 여부 확인

## 작업 전환 시 (CRITICAL)

**새 이슈 작업 시작 시 "이미 안다"고 가정하지 말 것**

```
이전 이슈 완료
    ↓
새 이슈 시작 전 STOP
    ↓
관련 스킬 재확인
    ↓
브랜치명, PR 형식 다시 확인
    ↓
작업 시작
```

## 지적사항 발생 시

1. **즉시** 해당 내용 기록
2. 같은 실수 반복 방지
3. **해당 영역 전체 재검토** 수행
   - 예: PR 양식 실수 → 모든 PR 점검
   - 예: 이슈 어사이니 누락 → 모든 이슈 점검
