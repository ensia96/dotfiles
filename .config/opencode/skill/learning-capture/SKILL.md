# Learning Capture

> 💡 **OPTIONAL SKILL**: 작업 과정에서 발견/결정/문제해결 발생 시 기록.
>
> **원칙**: 판단하지 말고 기록하라.

---

## 트리거

다음 상황 발생 시 기록:

| 상황 | 예시 |
|------|------|
| 문제 해결 직후 | 원인 파악, 해결 방법 발견 |
| 예상과 다른 결과 | API 동작, 설정 값, 에러 메시지 |
| 기술적 결정 | A vs B 선택, 구현 방식 결정 |
| 새로운 발견 | 시스템 구조, 숨겨진 동작 |

---

## 기록 원칙

| 원칙 | 설명 |
|------|------|
| **판단 금지** | "이건 기본이니까 스킵" 금지 |
| **전부 기록** | 사용자 지식 레벨 추측하지 않음 |
| **즉시 기록** | 맥락이 사라지기 전에 |
| **간결하게** | 형식은 최소화, 내용은 전부 |

---

## 형식

### 💡 {제목}

- **발견**: {무엇을 알게 되었는가}
- **맥락**: {어떤 상황에서}
- **활용**: {향후 어떻게 쓸 수 있는가}

---

## 예시

### 💡 K8s ServiceAccount 토큰 만료

- **발견**: K8s 1.24+에서 ServiceAccount 토큰이 자동 만료됨
- **맥락**: Dashboard 로그인 실패 원인 조사 중
- **활용**: 장기 토큰 필요 시 `expirationSeconds` 명시 또는 Secret 직접 생성

### 💡 Terraform state lock 충돌

- **발견**: `terraform force-unlock`으로 강제 해제 가능
- **맥락**: CI 중단 후 state lock 남아있어 apply 불가
- **활용**: lock ID 확인 후 force-unlock, 단 동시 실행 중이 아닌지 확인 필수

## before-report와의 관계

```
learning-capture (과정 중, 여러 번)
    │
    │  인사이트 (원재료)
    │
    ↓
before-report (완료 시, 1번)
    │
    │  지식 (정형화)
    │
    ↓
사용자에게 전달
```

- `learning-capture`: 즉시 기록, 간결한 형식
- `before-report`: 취합하여 구조화된 학습 포인트로 변환

---

## 금지 사항

| 금지 | 이유 |
|------|------|
| "사용자가 알 것 같아서 생략" | 지식 레벨 추측 금지 |
| "너무 기본적이라 스킵" | 기본의 기준이 사람마다 다름 |
| "나중에 정리하려고 안 적음" | 맥락 손실, before-report에서 취합 불가 |
