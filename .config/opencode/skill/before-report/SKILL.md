# Before Report

## 목적

작업 완료 후 사용자가 **지식과 간접 경험을 습득**할 수 있도록 학습 자료 제공.

---

## 서브 에이전트인 경우

**메인 모델에게 전달할 원재료 정리**

### 포함할 내용

```markdown
## 기술적 발견사항

### 사용한 기술/도구
- {도구명}: {용도}

### 발견한 사실
- {작업 중 알게 된 것}
- {예상과 달랐던 점}

### 겪은 문제와 해결
- 문제: {발생한 문제}
- 원인: {원인}
- 해결: {해결 방법}

### 유용한 명령어/코드
```bash
{작업 중 사용한 명령어}
```
```

### 원칙
- 사실만 기록 (해석/가공은 메인 모델이)
- 간결하게
- 기술적 세부사항 포함

---

## 메인 모델(Sisyphus)인 경우

**서브 에이전트 보고 + 직접 작업 내용을 종합하여 학습 자료로 가공**

### 제공 형식

```markdown
## 학습 포인트

### 이번 작업에서 배울 수 있는 것

**1. {주제}**
- 개념: {간단한 설명}
- 적용: {이번 작업에서 어떻게 적용됐는지}
- 참고: {공식 문서 또는 추가 자료}

### 기술적 결정과 이유

| 결정 | 이유 | 대안 |
|------|------|------|
| {선택한 방법} | {왜 선택했는지} | {고려했던 다른 방법} |

### 주의사항 / 함정

- {겪은 문제와 해결 방법}
- {다음에 비슷한 작업 시 주의할 점}

### 유용한 명령어

```bash
{유용한 명령어 모음}
```
```

### 원칙
- **사용자 관점**: "이걸 알면 뭐가 좋은가?"
- **맥락 제공**: "왜 이렇게 했는가?"
- **재사용 가능**: "다음에 어떻게 활용할 수 있는가?"
- **간결하게**: 핵심만 전달

---

## 제공 시점

| 상황 | 동작 |
|------|------|
| 이슈/PR 완료 | 학습 포인트 제공 |
| 복잡한 문제 해결 후 | 문제-원인-해결 정리 |
| 새로운 기술/도구 사용 | 개념과 사용법 정리 |
| 사용자 요청 시 | 요청 범위에 맞게 제공 |
