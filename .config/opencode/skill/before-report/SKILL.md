# Before Report

> **원칙**: 기록 → 검토 → 보고. 누락 없이.

## 목적

1. **작업 중 기록**: 발견/결정/문제해결 즉시 기록
2. **완료 전 검토**: 규칙 준수 여부 점검
3. **최종 보고**: 사용자에게 정리된 정보 전달

## 1. 작업 중 기록

### 트리거

| 상황 | 예시 |
|------|------|
| 문제 해결 직후 | 원인 파악, 해결 방법 발견 |
| 예상과 다른 결과 | API 동작, 설정 값, 에러 메시지 |
| 기술적 결정 | A vs B 선택, 구현 방식 결정 |
| 새로운 발견 | 시스템 구조, 숨겨진 동작 |

### 기록 형식

```markdown
### 💡 {제목}
- **발견**: {무엇을 알게 되었는가}
- **맥락**: {어떤 상황에서}
- **활용**: {향후 어떻게 쓸 수 있는가}
```

### 원칙

- 판단 금지 ("이건 기본이니까 스킵" 금지)
- 전부 기록 (사용자 지식 레벨 추측하지 않음)
- 즉시 기록 (맥락이 사라지기 전에)

## 2. 완료 전 검토

### 공통 체크리스트

```
[ ] 적용할 스킬을 모두 확인하고 적용했는가?
[ ] 네이밍 규칙을 따랐는가? (naming)
[ ] 커밋 메시지 형식을 따랐는가? (git-commit)
[ ] 민감 정보가 포함되지 않았는가?
```

### 서브에이전트 추가

```
[ ] 위임받은 범위 내에서 작업했는가?
[ ] MUST DO / MUST NOT DO 준수했는가?
[ ] 메인 모델에 전달할 정보가 충분한가?
```

### 메인 에이전트 추가

```
[ ] 서브에이전트 결과물이 컨벤션을 준수하는가?
[ ] 사용자의 원래 요청을 모두 충족했는가?
```

### 작업 전환 시

새 이슈 시작 전 관련 스킬 재확인. "이미 안다"고 가정하지 말 것.

## 3. 최종 보고

### 서브에이전트 → 메인 모델

```markdown
## 작업 결과
- 수행한 작업: {내용}
- 규칙 준수: {위반 여부}
- 기술적 발견: {작업 중 알게 된 것}
- 겪은 문제: {문제 → 원인 → 해결}
```

### 메인 모델 → 사용자

```markdown
## 작업 감사
- 이슈/PR: #{번호}
- 변경 파일/리소스 목록
- 규칙 준수 체크리스트
- 위반사항 및 시정 상태

## 학습 포인트
- {주제}: 개념, 적용, 참고
- 기술적 결정: {선택 → 이유 → 대안}
- 주의사항: {겪은 문제와 해결}
```

### 원칙

- 사용자 관점: "이걸 알면 뭐가 좋은가?"
- 맥락 제공: "왜 이렇게 했는가?"
- 재사용 가능: "다음에 어떻게 활용할 수 있는가?"

## 위반 발견 시

1. 시정 가능 → 직접 수정 또는 재지시
2. 시정 불가 → 사유 기록 후 사용자에게 보고
3. **모든 시정 완료 후** 최종 보고

## 제공 시점

| 상황 | 동작 |
|------|------|
| 이슈/PR 완료 | 작업 감사 + 학습 포인트 |
| 복잡한 문제 해결 후 | 문제-원인-해결 정리 |
| 새로운 기술/도구 사용 | 개념과 사용법 정리 |
