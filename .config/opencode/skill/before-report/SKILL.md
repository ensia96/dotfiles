# Before Report

## 목적

1. **작업 감사**: 규칙 준수 여부 점검 (self-audit 통합)
2. **학습 자료**: 사용자가 지식과 간접 경험을 습득할 수 있도록 정리

---

## 역할 구분

| 역할 | 담당 | 책임 |
|------|------|------|
| **메인 모델 (감독자)** | Sisyphus | 작업 위임, 결과 검토, 최종 보고 |
| **서브에이전트 (작업자)** | explore, frontend 등 | 실제 작업 수행, 결과 보고 |
| **사용자 (팀장)** | 인간 | 최종 결정, 방향 승인 |

---

## 서브에이전트인 경우

**메인 모델에게 전달할 원재료 정리**

### 포함할 내용

```markdown
## 작업 결과

### 수행한 작업
- {작업 내용}

### 규칙 준수 여부
- [ ] 위임받은 범위 내 작업
- [ ] MUST DO / MUST NOT DO 준수
- [ ] 네이밍 규칙 준수

### 기술적 발견사항
- {작업 중 알게 된 것}
- {예상과 달랐던 점}

### 겪은 문제와 해결
- 문제: {발생한 문제}
- 원인: {원인}
- 해결: {해결 방법}

### 유용한 명령어/코드
```bash
{작업 중 사용한 명령어}
```
```

### 원칙
- 사실만 기록 (해석/가공은 메인 모델이)
- 간결하게
- 기술적 세부사항 포함

---

## 메인 모델(Sisyphus)인 경우

**서브 에이전트 보고 + 직접 작업 내용을 종합하여 최종 보고**

### 1. 작업 감사 (Self-Audit)

```markdown
## 작업 감사

### 작업 요약
- 이슈: #{번호} {제목}
- PR: #{번호}
- 변경 파일/리소스 목록

### 규칙 준수 체크리스트
- [ ] 브랜치명: `{issue}/{worker}/{YYYY-MM-DD}` 형식 준수
- [ ] PR 제목: `[#{이슈}] {작업자} ({날짜})` 형식 준수
- [ ] 이슈 연결: `Closes #` 또는 `gh issue develop` 사용
- [ ] Merge 방식: 일반 merge (squash 아님)
- [ ] 네이밍 규칙: naming 스킬 준수
- [ ] 수동 작업: 이슈에 기록 완료

### 위반사항 (있는 경우)
| 위반 | 시정 상태 |
|------|----------|
| {내용} | {완료/불가/진행중} |

### 미해결 과제
- {내용}
```

### 2. 학습 포인트

```markdown
## 학습 포인트

### 이번 작업에서 배울 수 있는 것

**1. {주제}**
- 개념: {간단한 설명}
- 적용: {이번 작업에서 어떻게 적용됐는지}
- 참고: {공식 문서 또는 추가 자료}

### 기술적 결정과 이유

| 결정 | 이유 | 대안 |
|------|------|------|
| {선택한 방법} | {왜 선택했는지} | {고려했던 다른 방법} |

### 주의사항 / 함정

- {겪은 문제와 해결 방법}
- {다음에 비슷한 작업 시 주의할 점}

### 유용한 명령어

```bash
{유용한 명령어 모음}
```
```

### 원칙
- **사용자 관점**: "이걸 알면 뭐가 좋은가?"
- **맥락 제공**: "왜 이렇게 했는가?"
- **재사용 가능**: "다음에 어떻게 활용할 수 있는가?"
- **간결하게**: 핵심만 전달

---

## 검토 프로세스

```
서브에이전트: 작업 완료 후 결과 반환
    ↓
메인 모델: 규칙 준수 여부 검토
    ↓
┌─────────────────────────────────────┐
│ 정상 → 사용자에게 결과 보고         │
│ 위반 → 시정 지시 또는 직접 수정     │
│        (사용자 보고 전 해결 필수)   │
└─────────────────────────────────────┘
```

## 위반 발견 시

1. 시정 가능 여부 판단
2. 시정 가능 → 직접 수정 또는 서브에이전트에 재지시
3. 시정 불가 → 사유 기록 후 사용자에게 보고
4. **모든 시정 완료 후** 사용자에게 최종 보고

## 사용자 판단 필요 여부

| 구분 | 메인 모델 처리 | 사용자 판단 필요 |
|------|---------------|-----------------|
| 시정 가능한 위반 | 직접 수정 후 보고 | - |
| 시정 불가한 위반 | 기록만 | 영향 크면 보고 |
| 규칙 해석 모호 | - | 판단 요청 |
| 되돌릴 수 없는 작업 | - | 사전 승인 필요 |

---

## 제공 시점

| 상황 | 동작 |
|------|------|
| 이슈/PR 완료 | 작업 감사 + 학습 포인트 제공 |
| 복잡한 문제 해결 후 | 문제-원인-해결 정리 |
| 새로운 기술/도구 사용 | 개념과 사용법 정리 |
| 사용자 요청 시 | 요청 범위에 맞게 제공 |
