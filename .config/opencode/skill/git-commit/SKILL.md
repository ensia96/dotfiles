# Git Commit

## 형식

```
{type}: {message}
```

## 작성 원칙

- **한글 필수**: type만 영문, message는 한글
- **자기 완결적**: commit title만으로 변경 내용 파악 가능
- **명확성 우선**: message만으로 대상이 불분명하면 대상까지 명시

## Types

| Type | 용도 |
|------|------|
| `init` | 새로운 것 추가 |
| `feat` | 기존 것 변경 |
| `fix` | 버그 수정 |
| `docs` | 문서/주석 |
| `chore` | 동작에 영향 없는 변경 |

## Message 작성

| 상황 | 작성 방식 |
|------|----------|
| 대상이 명확함 | `{type}: {동작}` |
| 대상이 불명확함 | `{type}: {대상} {동작}` 또는 `{type}: {대상}의 {동작}` |

```
# 대상이 명확한 경우 (단일 파일, 명확한 기능)
init: 로그인 페이지 추가
fix: 무한 루프 해결

# 대상이 불명확한 경우 (여러 파일, 범위가 넓음)
feat: git-commit 스킬에 양식/내용 구분 기준 추가
fix: git-commit 스킬 '새 저장소' 표현 수정
```

## 저장소별 스타일

**커밋 전**: 기존 커밋 이력 확인

```bash
git log --oneline -20
```

| 구분 | 기준 |
|------|------|
| **양식** | 현재 스킬 (`{type}: {message}`) |
| **내용** | 기존 이력의 표현 방식, 어휘 참조 |

## 커밋 전 확인

- [ ] 한글로 작성했는가? (type 제외)
- [ ] message만으로 "뭘 했는지" 알 수 있는가?
- [ ] 민감 정보가 포함되지 않았는가?

## 금지

- 영문 message
- 대상 불명확한 모호한 message
